JavaScript: Your Next Steps

Normally in the last week of class, I like to talk about where you can continue your JavaScript education. This quarter, due to external events, I didn't get around to that. So I'd like to take this opportunity, in these last few minutes together, to talk about good resources to check out if you think JavaScript is a language you'd like to continue studying.

Most JavaScript programmers will point to a single book when asked about their professional development: JavaScript: The Good Parts by Douglas Crockford. Crockford did as much as anyone else to show the ways that JavaScript could be a serious, featureful programming language if examined with a critical eye. His book is extremely dense and slow to read, even at only 140 pages, and it is often wrong given what we know now about the ways that JavaScript is actually executed, but it is always wrong in interesting and educational ways.

Axel Rauschmeyer recently released an excellent book on the language itself called Speaking JavaScript, available online for free. If you want a book that will cover more of the browser and a number of JavaScript libraries, I highly recommend David S. Flanagan's JavaScript: The Definitive Guide, both as a text and as a way to kill small vermin.

One of the best ways to continue learning about this language is to read the source code of projects that use it, since it has a strong open-source community around it. Reading code is also itself an important skill, since it's vital when working on a team with other developers who may not write the same way you do. Here are some starting points:

  1. The jQuery library itself is fairly readable.
  2. Paul Irish on the Chrome team has a great set of presentations on things he learned by reading the jQuery source.
  3. Backbone and Underscore both offer "literate" documentation for their source.
  4. I think the source code for Caret is reasonably well-documented and clean.

Finally, I would strongly suggest anyone who is working in front-end development and design to check out NodeJS, the server-side JavaScript runtime. A lot of the newest, most interesting tools for web development are written for Node, including LESS, Grunt, Bower, Yeoman, Atom, Brackets, and Light Table.
